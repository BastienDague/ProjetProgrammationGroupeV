\section{Introduction}
\subsection{A-SABR}
Notre projet de programmation porte sur la librairie ASABR, portée par Mr Olivier De Jonckère du LIRMM.

Le réseau de communication spacial repose sur le \gls{dtn}. Avec ce type de protocole, le routage s'effecture par des algorithmes de type \gls{sabr}.\\

La librairie A-SABR implémente donc ce type d'algorithme en proposant une grande modularité et une bonne performance.\cite{sabr}
C'est également pour suivre ce choix de performance que celle-ci est programmé en Rust, language connue pour son compilateur stricte et sa fine gestion de la mémoire.

si Err, on retourne l'erreur et on l'affiche dans l'App Event.

Etant basé sur A-SABR, il faut régulièrement faire de petite modif dûe à des refactorisation de fonction ou de
l'ajout de Result remplaçant des Options dans la librairie. \\

\paragraph{Contact,Hybrid et Node parenting :}~\\\\
Contact,Hybride et Node sont 3 variantes de l'algorithme de Dijkstra qui sont utilisés pour le routage :\\

Node est le plus simple, il explore le \gls{mg} crée noeud par noeud, et ne maintient qu'une seule route optimal.
Il trouve le chemin optimal selon la métrique choisie mais s'arrête dès que la destination est atteinte en mode Path,
contrairement aux autres variantes qui continuent d'explorer.\\

Contact explore lui le \gls{mg} par contact plutôt que par noeud, il va utiliser une "Work Area", permettant d'explorer les contacts directement sans passer par la logique nœud par nœud, 
ce qui peut trouver des routes avec moins de saut dans certains cas.\\

Hybrid est la plus complète mais également la plus complexe, il va maintenir une liste de potentiel route optimal
tous le long du parcours, puis, grâce au fonction "can\_retain" et "must\_prune", va décider laquelle garder.\\

Chacunes de ces variantes possédent les 3 mêmes macros :
\begin{itemize}
    \item Path : Chemin simple vers une destination, s'arrête dès qu'elle est atteinte.
    \item PathExcl : Si l'on veut exclure un noeud précis.
    \item TreeExcl : Explore l'entièreté du \gls{mg} avec exclusion.\\
\end{itemize}

Les tests sonts simples, on créer plusieurs \gls{mg} auquel on applique chacunes des macros,
on utilise le manager le plus simple possible pour ne pas rendre le test trop lourd, on applique en dernier la méthode Hop (le moins de saut) ou la méthode SABR (le plus rapide) 
pour avoir le plus de données de test possible.

Les résultats attendues sont le nombre de saut réalisés, et le temps d'arrivée.
Pour les cas d'exclusion, on donne comme objectif par exemple le noeud 3 en excluant le noeud 2 et l'on doit avoir en retour un échec du au noeud exclus.\\

Dans un soucis de lisibilité et de maintienabilité du code, toute les fonctions de création d'objets (noeud,contact,bundle,multigraph)
sont réunis dans un fichiers à part (test\_helpers). De cette manière, seul les tests sont présents dans leur fichiers cibles. \\

Prenons un test en particulier et décryptons le : \label{testPathFinding} \ref{CodetestPathFinding} (Cliquer sur le numéro pour être envoyé à l'annexe correspondant) \\

Nous avons donc une première fonction qui va créer le \gls{mg} auquel on va applique nos tests, le multigraph créer possèdents
5 noeuds qui ont un manager neutre (qui ne fait rien) pour faire au plus simple et de 5 contacts.

Exemple : le premier contact émet du noeud 0 au noeud 1, début du contact à 0 et fin à 10, 
débit de 1 unité/seconde et aucun délai de propagation.

Ensuite nous créeons un \gls{bundle} à destination du noeud 4, sans priorité et sans taille précise.

On contruit les deux chemin, un avec la méthode SABR, l'autre avec la méthode Hop, 
puis nous lui appliquons notre fonction de test.\\

\begin{itemize}
    \item Hop : On attend 3 sauts en 50 unité de temps.
    \item SABR : On attend 4 sauts en 50 unité de temps.\\
\end{itemize}

Le fait que le nombre de saut diverge entre Hop et SABR permet une plus grand couverture pour detecter une casse de la librarie.\\

    pub fn update(&mut self, path:String, algo: &str){
        match PredictionConfig::try_init(path.clone(), algo){
            Ok(update_config) => {
                self.a_sabr = ASabrInitState::Enabled(update_config);
                self.notify_observers(ChatAppEvent::Info(format!("Update done with : {path} and  {algo}")));
            }
            Err(error) => {
                self.a_sabr = ASabrInitState::Error(error.to_string());
                self.notify_observers(ChatAppEvent::Info("Update failed".to_string()));
            }
        }
    }

pub fn try_init(cp_path: String, algo : &str) -> io::Result<Self> {
        let (nodes, contacts) = IONContactPlan::parse::<NoManagement, EVLManager>(&cp_path)?;

        let nodes_length = nodes.len();
        let contacts_length = contacts.len();

        let node_index_map: HashMap<String, NodeID> = nodes
            .iter()
            .enumerate()
            .map(|(index, node)| (node.get_node_name().to_string(), index as NodeID))
            .collect();

        let router_box = build_generic_router::<NoManagement, EVLManager>(
            algo,
            nodes,
            contacts,
            None,
        ).expect("Failed to build router");

        let router: Box<dyn Router<NoManagement, EVLManager> + Send + Sync> =
            unsafe { std::mem::transmute(router_box) };
        // in seconds
        let cp_start_time = DTChatTime::now().timestamp_millis() as f64 / 1000.0;

        Ok(PredictionConfig {
            ion_to_node_id: node_index_map,
            router,
            cp_start_time,
            nodes_length,
            contacts_length,
        })
    }

fn test_exemple_2() -> Result<(), ASABRError> {
        let mg = exemple_2_graph()?;

        let mut algo_hop = NodeParentingPath::<NoManagement, EVLManager, Hop>::new(mg.clone());
        let mut algo_sabr = NodeParentingPath::<NoManagement, EVLManager, SABR>::new(mg.clone());

        let bundle = make_bundle(4, 0, 0.0, 1000.0);

        let res_hop = algo_hop
            .get_next(0.0, 0, &bundle, &[][..])
            .expect("Routing Failed !");

        assert_time_hop(&res_hop, 4, 50.0, 3, "Hop");

        let res_sabr = algo_sabr
            .get_next(0.0, 0, &bundle, &[][..])
            .expect("Routing Failed !");

        assert_time_hop(&res_sabr, 4, 50.0, 4, "SABR");

        Ok(())
    }
